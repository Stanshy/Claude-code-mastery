# 02-3 Agent = Model + Harness（預告版）

> Agent 的真正能力不只來自模型本身，而是模型加上周圍整套控制系統的總和。

---

## 學習目標

- 理解 Agent = Model + Harness 這個核心公式的含義
- 區分模型（Model）與 Harness 各自負責的職責
- 了解為什麼優化 Harness 比更換模型更具實際效益
- 建立對後續模組（Hooks、Subagents、Memory）的整體認知框架

## 正文

### 一句話定義

Agent = Model + Harness。

模型是引擎，Harness 是方向盤和煞車。引擎再強，沒有方向盤到不了目的地；沒有煞車，跑得越快越危險。這個等式不是隱喻，而是工程事實：一個 Agent 系統的實際表現，由這兩個部分共同決定，缺一不可。

### 為什麼這個概念重要

有一個真實案例值得深思。LangChain 的 Coding Agent 在 TerminoBench 排行榜上，從 30 名以外一路衝進前 5 名。底層模型一行沒換，工程師只動了三樣東西：系統提示的結構、工具的配置方式、以及中間件的鉤子邏輯。

這說明了一件事：模型不變，只優化周圍系統，就能帶來數量級的提升。

許多人直覺認為「模型更強，Agent 就更好」，因此把所有精力放在選模型、換模型上。這個想法並非錯誤，但它只答對了一半。Harness 才是決定日常輸出品質的關鍵變數，也是工程師真正能掌控的部分。

### 沒有 Harness 會發生什麼

以下這張表格是這章的核心。它對比了同一個模型，在有無 Harness 保護下的實際差異。

| 面向 | 沒有 Harness | 有 Harness |
|------|-------------|-----------|
| 檔案安全 | AI 常亂改不該改的檔案 | 敏感檔案被 Hook 保護，改不了 |
| 任務完成度 | 忘記跑測試就說完成了 | Stop Validator 強制驗證，沒通過不能停 |
| 一致性 | 每次結果不一致，同樣指令不同輸出 | 規則由工具執行，不依賴模型自覺 |
| 規則遵守 | 規則寫在 prompt 裡但 AI 時遵時不遵 | 確定性工具強制執行，100% 遵守 |
| 進化能力 | 每次犯同樣的錯 | 系統持續從錯誤中學習，越用越穩 |

結論很直接：模型決定「能力上限」，Harness 決定「實際輸出品質」。一個沒有 Harness 的強模型，在生產環境中往往不如一個有完善 Harness 包覆的普通模型來得可靠。

### Harness 在 Claude Code 中的對應

「Harness」是一個抽象概念，在 Claude Code 的具體實作中，它對應到六個不同的機制。以下是完整的對照表，以及你會在哪個模組學到它。

| Harness 概念 | Claude Code 對應 | 你會在哪裡學 |
|-------------|-----------------|-------------|
| 上下文架構 | CLAUDE.md + Skills | 模組 2、3 |
| 架構約束 | Hooks（PreToolUse / PostToolUse） | 模組 3 |
| 自驗證循環 | Stop Hook Validator | 模組 3 |
| 上下文隔離 | Subagents | 模組 4 |
| 熵管理 | Auto Memory + 踩坑閉環 | 模組 4 |
| 可拆卸性 | 模組化 Hook / Skill / Agent | 模組 6 |

這六個機制不是各自獨立的工具，而是一套協同運作的系統。上下文架構告訴模型「你是誰、你能做什麼」；架構約束確保它不會越界；自驗證循環讓它在宣稱完成前先自我審查；上下文隔離讓複雜任務不互相干擾；熵管理讓系統在長期使用中保持穩定；可拆卸性則確保整套系統能夠持續演進而不腐化。

### 預告：觀念之後是實作

這章只是建立觀念，目的是讓你在接觸後續所有工具時，腦海中始終有一張地圖。當你在模組 3 設定第一個 PreToolUse Hook 時，你會知道自己正在做的是「安裝 Harness 的煞車系統」；當你在模組 4 拆分 Subagent 時，你會知道自己正在處理的是「上下文隔離」問題。

模組 6 會完整展開 Harness 的七大設計原則，並以 AI Dev Assistant 專案的完整演進過程作為案例，帶你看清楚一個從零開始的 Agent 系統，是如何一步步長出它的 Harness。

## 關鍵重點

- Agent = Model + Harness，兩者缺一不可，不能只看模型能力
- 模型決定能力上限，Harness 決定實際輸出品質
- 優化 Harness（提示結構、工具配置、鉤子邏輯）可以在不換模型的前提下帶來數量級提升
- Harness 的本質是將不穩定的「模型自覺」轉化為確定性的「系統執行」
- Claude Code 的 Hooks、Skills、Subagents、CLAUDE.md 都是 Harness 的具體實作形式
