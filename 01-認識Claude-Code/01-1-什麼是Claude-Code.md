# 01-1 什麼是 Claude Code

> Claude Code 是一個在終端機中運行的自主執行代理，而不是聊天機器人或 IDE 外掛。

---

- 用一句話精確說明 Claude Code 的定義與類型

- 親手執行一次 Agent Loop，觀察 Claude Code 自主完成任務的過程

## 正文

### 1. 一句話定義

Claude Code 是 Anthropic 推出的 **agentic coding CLI**（自主代理命令列工具）。

這句話有三個關鍵字：

- **agentic**：它不等待你逐步指揮，它會主動決策、探索、修正
- **coding**：它的核心能力是處理程式碼與軟體工程任務
- **CLI**：它在終端機中運行，不依附任何特定 IDE

它能做到以下四件事，而且全部自主完成：讀取整個 codebase、跨多個檔案進行編輯、執行系統命令（如測試或建構）、驗證自己的修改是否正確。

這與 IDE 外掛（如 GitHub Copilot）的「補全下一行」，或是 ChatGPT 的「生成程式碼片段給你貼上」，在工作方式上有根本的不同。

---

### 2. 代理循環（Agent Loop）

Claude Code 的工作模型是一個持續執行的循環，而不是單次回應：

```
讀取 → 判斷 → 執行 → 驗證
 ↑                        |
 └────────────────────────┘
      （直到任務完成）
```

**讀取**：Claude Code 掃描專案結構、讀取相關檔案、理解你的需求。
**判斷**：決定接下來要做什麼，例如應該修改哪個檔案、呼叫哪個命令。
**執行**：實際進行檔案編輯、執行測試、呼叫 shell 命令。
**驗證**：確認執行結果是否符合預期，若不符合則重新進入循環。

這個循環會一直運行，直到任務完成，或是遇到需要人類做出決策的分岔點（例如：是否刪除一個你沒有明確授權刪除的檔案）。

這個設計的意義在於：你不需要把任務切成每一步再一步步交給 Claude。你可以描述最終目標，Claude 自行規劃路徑並執行到底。代價是：你需要理解它在做什麼，否則難以判斷它是否走偏。

---

### 3. 與其他 AI 編碼工具的差異

以下對比不是為了評分，而是幫你判斷在什麼情況下應該選哪個工具（截至 2026-03 的公開方案，規格與價格可能變動）：

| 維度 | Claude Code | Cursor | GitHub Copilot | Windsurf |
|------|-------------|--------|----------------|----------|
| 類型 | 終端 CLI | IDE（VS Code fork） | IDE 外掛 | IDE |
| 工作方式 | 自主代理循環 | Agentic + 補全 | 補全 + Chat | Flow 模式 |
| 多檔操作 | 自動跨檔 | 自動跨檔 | 需手動指定 | 自動跨檔 |
| 命令執行 | 可跑測試、部署 | 可跑命令 | 不可 | 可跑命令 |
| 上下文窗口 | 200K（最大） | 128K | 視模型而定 | 128K |
| 最強場景 | 複雜架構、自主多步驟任務 | 日常 IDE 內編碼 | Boilerplate 補全 | 意圖推斷流程 |
| 價格起點 | $20/月（Pro） | $20/月（Pro） | $10/月 | $15/月 |

Claude Code 的 200K 上下文窗口代表它可以同時理解更大的 codebase，不需要你手動選取「相關檔案」。這在處理中大型專案時會帶來明顯差異。

---

### 4. 適用場景與不適用場景

**Claude Code 最適合以下場景：**

- **建構新功能**：描述需求，Claude 自主規劃實作路徑並完成
- **修復 bug**：貼上錯誤訊息，Claude 定位根因、修復、執行測試驗證
- **Code review 與重構**：指定檔案或範圍，Claude 分析並提出改動
- **測試撰寫**：根據現有程式碼自動生成對應的測試案例
- **Git 工作流**：commit 訊息生成、PR 描述、branch 管理
- **文件維護**：根據程式碼現況更新 README 或 API 文件

**以下場景不建議使用 Claude Code：**

- **即時 autocomplete**：你在打字時需要逐字補全，Copilot 的延遲和整合更適合這個需求
- **視覺化 UI 開發**：需要頻繁在瀏覽器預覽結果、即時調整樣式，CLI 工具缺乏這個回饋迴路
- **極小的單行修改**：若你只是需要改一個變數名稱，啟動 Agent Loop 的開銷不值得

判斷原則：如果任務需要「多個步驟、多個檔案、且步驟之間有依賴關係」，Claude Code 是合適的工具。如果任務是「單一位置的點對點修改」，直接編輯或用補全工具更快。

---

### 5. 五層進化模型預覽

大多數人第一次接觸 Claude Code 時，只是把它當成一個更聰明的聊天視窗。這門課的目標是帶你系統性地走過以下五個層級：

| 層級 | 名稱 | 你與 Claude Code 的關係 | 對應模組 |
|------|------|------------------------|---------|
| 第一層 | 對話式 | 你輸入提示，Claude 回應執行 | 模組 1 |
| 第二層 | 配置式 | CLAUDE.md 引導行為，跨對話記憶保留 | 模組 2 |
| 第三層 | 自動化 | Hooks 強制執行規則，Skills 編碼工作流 | 模組 3 |
| 第四層 | 編排式 | 自訂 Agents 各司其職，分工完成複雜任務 | 模組 4 |
| 第五層 | 自主式 | Agent 團隊獨立運作並互相驗證結果 | 模組 5 |

第一層的門檻很低，這也是為什麼大多數人停在這裡——它已經夠用了。但第一層的 Claude Code 沒有記憶、沒有規則、沒有可複用的工作流，每次對話都從零開始。

第三層以上才是 Claude Code 真正的工程價值：你建構的不只是一次性的輔助，而是一個可重複、可維護、可擴展的自動化系統。

## 實作範例：觀察 Agent Loop 執行一次完整任務

> 主案例延續：在 AI Dev Assistant 專案的第一步，先確認你能成功觸發 Agent Loop。

**目標**：讓 Claude Code 自主建立一個 TypeScript 函數，親眼觀察 Agent Loop 的執行過程。

**前置條件**：已安裝 Claude Code 並完成認證（若尚未安裝，請先閱讀 01-2）。

**步驟**：

1. 打開終端機（macOS 用 Terminal，Windows 用 Git Bash 或 WSL）
2. 建立一個空的工作資料夾：
   ```bash
   mkdir demo && cd demo
   ```
3. 啟動 Claude Code：
   ```bash
   claude
   ```
4. 在 Claude Code 的提示字元中輸入以下指令：
   ```
   建立一個 TypeScript 檔案 add.ts，裡面有一個 add 函數，接收兩個數字並回傳相加結果
   ```
5. 觀察 Claude 的行為。你會看到它：
   - 決定需要建立哪個檔案
   - 生成程式碼並寫入磁碟
   - 回報完成結果
6. Claude 完成後，在另一個終端機視窗執行：
   ```bash
   cat add.ts
   ```
   確認檔案確實存在且內容正確。

**預期結果**：

- `add.ts` 自動出現在 `demo/` 資料夾中
- 檔案內容包含一個接收兩個 `number` 參數並回傳 `number` 的 `add` 函數
- 整個過程你只輸入了一句自然語言，沒有手動建立檔案或貼上程式碼

**常見錯誤**：

- `command not found: claude`：Claude Code 尚未安裝，或安裝路徑不在 PATH 中。請參考 01-2 的安裝章節。
- `You need to authenticate`：尚未登入 Anthropic 帳號。執行 `claude` 後按照終端機提示完成 OAuth 認證流程。
- Claude 詢問是否可以建立檔案但等待確認：預設設定下 Claude 在某些操作前會請求許可，輸入 `y` 確認即可。

注意：這個範例刻意省略了測試、git commit、以及任何驗證步驟。它的唯一目的是讓你親眼看到「自然語言輸入 → Agent 自動執行 → 結果出現在檔案系統」這個核心循環。完整的專案建構流程，包含測試與版本控制，從 01-2 開始介紹。

## 關鍵重點

- Claude Code 是 **agentic coding CLI**，它的工作單位是「任務」，不是「回應」
- Agent Loop（讀取 → 判斷 → 執行 → 驗證）是它與補全型 AI 工具的本質差異
- 200K 上下文窗口讓它可以理解整個 codebase，而不只是你貼過來的片段
- 適合多步驟、多檔案、需要執行驗證的任務；不適合即時補全和單點小修改
- 大多數人停在第一層（對話式）；這門課的目標是帶你到第三層以上
