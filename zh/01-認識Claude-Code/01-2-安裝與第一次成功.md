# 01-2 安裝與第一次成功
> 選對安裝方式，五分鐘內完成設定，讓 Claude Code 建立你的第一個 TypeScript 專案骨架。

## 學習目標
- 能說明三種安裝方式的差異，並選擇適合自己環境的方式
- 完成 OAuth 認證流程，通過 `claude doctor` 健康檢查
- 使用 Claude Code 建立符合規格的 TypeScript 專案骨架
- 執行 `npm test` 和 `npx tsc`，看到零錯誤的成功輸出

---

## 正文

### 1. 為什麼安裝方式很重要

大多數工具的安裝方式只是手段，但 Claude Code 的安裝方式直接決定了三件事：**更新機制**、**權限模型**、**版本穩定性**。

用 `sudo npm install -g` 安裝會把全域套件寫入系統目錄，之後每次更新都需要 sudo，權限問題會持續困擾你。用 npm 安裝雖然方便，但不會自動更新，你需要自己追蹤新版本。

原生安裝器是 Anthropic 官方維護的發行通道，負責自動更新，並且不依賴你本機的 Node.js 環境。這是日常開發最穩定的選擇。

---

### 2. 三種安裝方式

| 方式 | 指令 | 自動更新 | 適用情境 |
|------|------|---------|---------|
| 原生安裝器（推薦） | macOS/Linux：`curl -fsSL https://claude.ai/install.sh \| bash`<br>Windows PowerShell：`irm https://claude.ai/install.ps1 \| iex` | 是 | 日常開發 |
| npm | `npm install -g @anthropic-ai/claude-code` | 否 | 需要鎖定特定版本的 CI 環境 |
| Homebrew | `brew install --cask claude-code` | 否 | macOS 且偏好統一用 brew 管理工具 |

**npm 安裝的前提條件**：本機需要 Node.js 18.x 以上。執行 `node --version` 確認版本。如果版本不符，透過 [nvm](https://github.com/nvm-sh/nvm)（macOS/Linux）或 [nvm-windows](https://github.com/coreybutler/nvm-windows) 安裝正確版本，不要直接修改系統 Node.js。

**不要用 sudo 執行 npm install -g**。sudo 會讓全域套件寫入 root 擁有的目錄，導致後續所有 npm 操作都需要提權，這是一個難以逆轉的環境污染問題。

---

### 3. 認證流程

安裝完成後，第一次執行 `claude` 時，終端機會輸出一個認證 URL，並自動開啟瀏覽器。

認證頁面支援以下帳號類型：
- **Claude Pro / Max**：個人訂閱帳號
- **Teams / Enterprise**：組織帳號，透過管理員開通
- **Anthropic Console（API Key）**：直接使用 API key，適合自動化場景

完成瀏覽器授權後，認證 token 會儲存在本機的設定檔中。之後啟動 `claude` 不需要重複登入，除非 token 過期或你手動執行 `claude logout`。

---

### 4. 驗證安裝

安裝與認證完成後，執行以下兩個指令確認環境正常：

```bash
claude --version
```

輸出範例：`claude 1.x.x`，確認 CLI 可以正常執行。

```bash
claude doctor
```

`claude doctor` 會檢查認證狀態、網路連線、以及相依環境是否符合需求。所有項目顯示綠色勾號，才算完整通過。如果有項目失敗，`doctor` 會給出具體的修復建議。

---

### 5. IDE 整合（選用）

Claude Code 本質上是一個終端機工具，不需要 IDE 就能完整運作。如果你偏好在 IDE 中使用：

- **VS Code**：在擴展市場搜尋 "Claude Code" 並安裝，安裝後可以在編輯器側邊欄直接開啟 Claude Code 對話。
- **JetBrains（IntelliJ、WebStorm 等）**：在插件市場搜尋 "Claude Code" 安裝。

兩種整合方式都只是提供一個內嵌的終端機介面，底層仍然是同一個 CLI 工具，行為完全一致。本課程的所有操作都以終端機為主，IDE 整合是個人偏好選項。

## 實作範例：建立 AI Dev Assistant 專案骨架
> 主案例延續：這是 AI Dev Assistant 專案的起點，後續所有章節都會在這個骨架上繼續開發。

**目標**：用 Claude Code 建立符合規格的 TypeScript 專案骨架，包含正確的 tsconfig、測試配置和 npm scripts。

**前置條件**：已完成 Claude Code 安裝，執行 `claude doctor` 全部項目通過。

**步驟**：

1. 在終端機建立專案目錄並進入：
   ```bash
   mkdir ai-dev-assistant && cd ai-dev-assistant
   ```

2. 啟動 Claude Code：
   ```bash
   claude
   ```
   看到 Claude 的提示符號後，代表已進入互動模式。

3. 輸入以下指令，讓 Claude 建立整個專案骨架：
   ```
   初始化一個 TypeScript 專案：
   - 用 npm init 建立 package.json
   - 安裝 typescript@5、jest@29、ts-jest、@types/jest、eslint@8、prettier@3 為 devDependencies
   - 建立 tsconfig.json（target: ES2022, module: NodeNext, strict: true）
   - 建立 src/index.ts（export 一個空的 main 函數）
   - 建立 tests/index.test.ts（測試 main 函數存在）
   - 在 package.json 加上 scripts: build, test, lint
   ```

4. 觀察 Claude 的執行過程。你會看到 Claude 依序執行 shell 指令、建立檔案、寫入內容。每個步驟 Claude 都會顯示它正在做什麼，以及為什麼這樣做。

5. Claude 完成後，不要關閉對話，先在另一個終端機視窗驗證結果：
   ```bash
   npm test
   ```
   預期輸出包含 `Tests: 1 passed`。

   ```bash
   npx tsc
   ```
   沒有任何輸出代表編譯成功，有輸出代表有型別錯誤需要修正。

6. 確認目錄結構：
   ```bash
   ls -la
   ```
   應該看到 `package.json`、`tsconfig.json`、`src/`、`tests/`、`node_modules/`。

**預期結果**：
- `package.json` 的 `devDependencies` 包含 typescript@5.x、jest@29.x、ts-jest、@types/jest、eslint@8.x、prettier@3.x
- `tsconfig.json` 設定 `target: "ES2022"`、`module: "NodeNext"`、`strict: true`
- `src/index.ts` 存在並 export `main` 函數
- `tests/index.test.ts` 存在並包含至少一個測試
- `npm test` 輸出 `1 passed`
- `npx tsc` 無錯誤輸出

**常見錯誤**：

- **`npm ERR! EACCES permission denied`**：全域 Node.js 目錄的擁有者是 root。解決方式是改用 nvm 管理 Node.js 版本，而不是修復現有的權限問題，因為修復後可能再次出現。

- **`Cannot find module 'ts-jest'`**：Claude 有時會在配置 jest 時忘記安裝 ts-jest。回到 Claude Code 對話，輸入「ts-jest 沒有安裝，請執行 npm install --save-dev ts-jest」，或手動執行 `npm install --save-dev ts-jest`。

- **`Cannot find type definition file for 'jest'`**：`tsconfig.json` 的 `compilerOptions.types` 陣列沒有包含 `"jest"`。確認 `@types/jest` 已安裝，並在 tsconfig.json 加上 `"types": ["jest"]`。

---

## 關鍵重點
- 原生安裝器是唯一支援自動更新的方式，日常開發優先選用
- 不要用 sudo 安裝任何 npm 全域套件，版本衝突和權限問題難以逆轉
- `claude doctor` 是確認環境完整性的標準工具，出現問題時先執行它
- Claude Code 不需要 IDE，終端機是完整的使用環境
- Claude 執行完成後，立即用 `npm test` 和 `npx tsc` 驗證結果，不要假設它一定正確

---

## 自我檢核
- 執行 `claude doctor`，所有項目是否都顯示通過？如果有失敗項目，你知道如何根據它的提示修復嗎？
- 在 `ai-dev-assistant` 目錄執行 `npm test`，輸出是否包含 `1 passed`？如果失敗，錯誤訊息指向哪個檔案或哪個設定？
- 執行 `npx tsc`，是否沒有任何錯誤輸出？如果有錯誤，錯誤訊息是否與 `tsconfig.json` 的設定有關？
